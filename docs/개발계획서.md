# 원고지 박살내기 — 개발 계획서

## Context

기획서(`기획서.md`)를 기반으로 `react-korean-manuscript@0.2.3` 패키지를 핵심으로 하는 한국어 원고지 교육 서비스를 구현한다. Next.js 16 단일 프로젝트에서 App Router(프론트엔드) + Route Handlers(백엔드)로 풀스택을 구성하고, Railway.com에 PostgreSQL과 함께 배포한다.

---

## 기술 스택 확정

| 항목 | 선택 | 버전 |
|------|------|------|
| 프레임워크 | Next.js (App Router) | 16.x |
| DB ORM | Drizzle ORM + drizzle-kit | 0.45 / 0.31 |
| DB 드라이버 | postgres.js | 3.x |
| 인증 | Auth.js (next-auth v5 beta) | 5.0.0-beta.30 |
| 상태관리 | Zustand | 5.x |
| 배포 | Railway.com | — |
| 핵심 패키지 | react-korean-manuscript | 0.2.3 |

**ORM 선택 이유 — Drizzle:** postgres.js 공식 통합, 타입 안전 집계 쿼리, `schema.ts`가 단일 진실 공급원, `drizzle-kit migrate` 한 줄로 Railway 자동 마이그레이션 가능.

## 핵심 설계

### 에디터 데이터 흐름

```
사용자 키 입력
    ↓
hidden <textarea> onChange
    ↓
editorStore.setText(newText)
    ├── useManuscriptEditor: buildCursorMaps(text) → setCursorCell, setCursorSide
    └── useRuleChecker (300ms debounce): runAllRules(text, level) → setErrorCells, setViolations
    ↓
<Manuscript text cursorCell cursorSide errorCells pageIndex
  onCellClick={(i) => uiStore.selectViolationCell(i)} />
    ↓
세션 종료 시: POST /api/sessions + POST /api/errors (배치)
```

### Zustand 스토어 3개 분리

- **editorStore**: `text`, `cursorCell`, `cursorSide`, `errorCells`, `pageIndex`, `violations` — `<Manuscript />` props 원천
- **sessionStore**: `sessionId`, `problemId`, `mode`, `status`, `elapsedSec`, `score`, `clickedCells`
- **uiStore**: `isSidePanelOpen`, `selectedViolationCellIndex`, `isScoreBoardVisible`

### 규칙 엔진 (`src/lib/rules/`)

레벨별 활성 규칙 배열로 단계적 추가:

```typescript
const RULES_BY_LEVEL = {
  1: [checkIndent, checkPunctuation, checkQuote, checkLineEnd],
  2: [...Lv1, checkSpacing],
  3: [...Lv2, checkNumberAlpha],
  4: [...Lv3, checkSpelling],
};

export function runAllRules(text: string, level: number): {
  violations: RuleViolation[];
  errorCells: Set<number>;
}
```

각 규칙 모듈은 `(ctx: RuleContext) => RuleViolation[]` 순수 함수. `buildFormattedCells()` + `buildCursorMaps()` 결과를 컨텍스트로 전달.

### DB 연결 싱글턴

```typescript
// src/lib/db/index.ts
const globalForDb = globalThis as unknown as { db: typeof db };
if (!globalForDb.db) {
  const client = postgres(process.env.DATABASE_URL!, {
    max: 5, idle_timeout: 20, connect_timeout: 10,
  });
  globalForDb.db = drizzle(client, { schema });
}
export const db = globalForDb.db;
```

### Railway 배포

```json
// railway.json
{
  "deploy": {
    "startCommand": "npm run db:migrate && next start",
    "healthcheckPath": "/api/health"
  }
}
```

`npm run db:migrate` = `drizzle-kit migrate` → 배포마다 자동 스키마 마이그레이션.

## 중요 구현 주의사항

1. **`<Manuscript />`는 렌더링 전용** — 실제 키 입력은 반드시 hidden `<textarea>`로 처리
2. **postgres.js 연결 풀 `max: 5`** — Railway Hobby 플랜 연결 수 제한 대응, globalThis 싱글턴 패턴 필수
3. **Auth.js v5는 `AUTH_SECRET`** — v4의 `NEXTAUTH_SECRET`와 변수명 다름 주의
4. **`output: 'standalone'`** — Railway 컨테이너 최소화 빌드, `next.config.ts`에 반드시 포함
5. **Railway `startCommand`에 마이그레이션 포함** — `"npm run db:migrate && next start"`로 무중단 자동 마이그레이션
