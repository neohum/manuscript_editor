## 1. 서비스 개요

### 1.1 서비스명 (가칭)
**원고지 박살내기** (Manuscript Trainer)

### 1.2 한 줄 소개
초등학교 5학년부터 성인까지, 200자 원고지 작성법과 한국어 맞춤법·띄어쓰기를 게임처럼 훈련하는 웹 에디터 서비스

### 1.3 핵심 문제 정의
| 문제 | 대상 |
|------|------|
| 원고지 쓰기 규칙(문장부호 위치, 들여쓰기, 행 첫칸 규칙 등)을 배울 기회가 없음 | 초등 고학년 ~ 중학생 |
| 국어 시험·논술에서 원고지 형식 오류로 감점 | 중·고교생 |
| 공공문서·공문서 작성 시 기본 맞춤법 실수가 잦음 | 성인 직장인 |
| 교사가 원고지 숙제를 내줘도 채점 기준이 주관적 | 교사·학부모 |

- 수행평가·논술 원고지 제출이 있음
- 규칙은 알지만 습관적으로 틀리는 패턴이 반복됨
- 오답 패턴 분석과 반복 훈련을 원함

### 페르소나 C — 성인 직장인 "박수연"
- 기안문·보고서 작성 시 맞춤법 지적을 받음
- 짧은 시간에 핵심만 교정·학습하길 원함
- 모바일보다는 데스크톱 사용

### 페르소나 D — 초등 교사 "최지영"

## 3. 핵심 기능 정의

### 3.1 원고지 에디터 (Editor Mode)
원고지 규격: **200자 / 가로 20칸 × 세로 10행 = 1페이지**

| 기능 | 설명 |
|------|------|
| 실시간 원고지 렌더링 | `<Manuscript />` 컴포넌트, `text` prop 실시간 업데이트 |
| 커서 위치 표시 | `cursorCell`, `cursorSide` prop으로 현재 입력 위치 시각화 |
| 공백 시각화 | `showSpaceMarks={true}` 로 띄어쓰기 위치를 점(·)으로 표시 |
| 행 번호 표시 | `showRowNums={true}` 로 학습 편의성 제공 |
| 다중 페이지 | 200자 초과 시 페이지 자동 분리, `pageIndex`로 개별 렌더링 |
| 오류 셀 하이라이트 | `errorCells` prop으로 맞춤법·형식 오류 위치를 붉게 표시 |

**원고지 캔버스 크기:** CSS 커스텀 프로퍼티로 가로 200%, 세로 100% 비율 유지
(`--manuscript-cell-bg`, `--manuscript-grid-line` 등으로 테마 커스터마이징)

### 3.2 원고지 규칙 검사 엔진
`buildFormattedCells()` 유틸을 기반으로 자체 규칙 레이어를 추가

| 규칙 카테고리 | 세부 규칙 |
|--------------|-----------|
| **들여쓰기** | 문단 첫 글자는 1칸 들여쓰기 |
| **문장부호** | 마침표·쉼표는 앞 글자와 같은 칸, 줄 맨 앞에 올 수 없음 |
| **따옴표** | `dblQuoteInit` / `sglQuoteInit`로 열고닫기 쌍 검사 |
| **행 끝 처리** | 행 마지막 칸에 여는 따옴표·여는 괄호 금지 |
| **숫자/영어** | 한 칸에 두 글자 병기 규칙 |
| **제목 행** | 상단 2~3칸 띄우기 |
| **띄어쓰기** | 조사, 의존명사, 보조용언 등 한국어 띄어쓰기 규칙 |
| **맞춤법** | 자주 틀리는 어휘 사전 기반 오류 감지 |

### 3.3 학습 모드 (Training Mode)
세 가지 서브모드로 구성:

#### A. 규칙 학습 (Rule Guide)
- 원고지 규칙을 인터랙티브 애니메이션으로 시각적 설명
- 규칙별 예시 문장을 원고지에 바로 렌더링하여 확인
- 단계별 진행: 들여쓰기 → 문장부호 → 따옴표 → 띄어쓰기 → 맞춤법

#### B. 받아쓰기 (Dictation Mode)
- 음성 또는 텍스트로 제시된 지문을 원고지에 입력
- 입력 완료 후 `errorCells`로 오류 위치 표시
- 오류 유형별 해설 제공

#### C. 교정 모드 (Correction Mode)
- 일부러 오류를 포함한 지문을 원고지에 표시
- 사용자가 오류 칸을 클릭(`onCellClick`)하여 교정 제출
- 정답 여부 즉시 피드백

### 3.4 수준별 커리큘럼

| 레벨 | 대상 | 내용 |
|------|------|------|
| **Lv.1 새싹** | 초5~6 | 원고지 기본 규칙 4가지, 쉬운 어휘 맞춤법 |
| **Lv.2 새순** | 중학생 | 문장부호 심화, 의존명사·보조용언 띄어쓰기 |
| **Lv.3 나무** | 고등학생 | 논술체 문장 원고지 형식, 고급 맞춤법 |
| **Lv.4 숲** | 성인 | 실무 문서 어휘, 자주 틀리는 경어법·외래어 표기 |

### 3.5 오류 분석 리포트
- 세션별 오류율, 오류 유형 분포 (파이 차트)
- 반복 실수 패턴 Top 5 시각화
- 교사용 뷰: 학생 전체 오류 현황 테이블 (CSV 내보내기)

### 5.4 데이터베이스 스키마 (PostgreSQL)

```sql
-- 사용자
CREATE TABLE users (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email       TEXT UNIQUE NOT NULL,
  name        TEXT NOT NULL,
  role        TEXT NOT NULL CHECK (role IN ('student', 'teacher')),
  level       INT NOT NULL DEFAULT 1,      -- 현재 학습 레벨 (1~4)
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 학급 (교사-학생 연결)
CREATE TABLE classrooms (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  teacher_id  UUID NOT NULL REFERENCES users(id),
  name        TEXT NOT NULL,
  invite_code TEXT UNIQUE NOT NULL,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE classroom_members (
  classroom_id UUID REFERENCES classrooms(id),
  student_id   UUID REFERENCES users(id),
  joined_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  PRIMARY KEY (classroom_id, student_id)
);

-- 문제 (커리큘럼)
CREATE TABLE problems (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  level        INT NOT NULL CHECK (level BETWEEN 1 AND 4),
  mode         TEXT NOT NULL CHECK (mode IN ('dictation', 'correction', 'free')),
  title        TEXT NOT NULL,
  content      TEXT NOT NULL,       -- 정답 텍스트 (받아쓰기 지문)
  error_hints  JSONB,               -- 교정 모드용 오류 위치·유형 힌트
  created_at   TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 학습 세션
CREATE TABLE learning_sessions (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id      UUID NOT NULL REFERENCES users(id),
  problem_id   UUID REFERENCES problems(id),  -- NULL = 자유 작성
  mode         TEXT NOT NULL,
  score        INT,                 -- 100점 만점
  duration_sec INT,                 -- 소요 시간(초)
  final_text   TEXT,                -- 최종 제출 원고
  created_at   TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 오류 로그 (세션별 개별 오류)
CREATE TABLE error_logs (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id   UUID NOT NULL REFERENCES learning_sessions(id),
  cell_index   INT NOT NULL,        -- 오류 발생 원고지 셀 위치
  error_type   TEXT NOT NULL,       -- 'indent' | 'punctuation' | 'spacing' | 'spelling' | 'quote'
  wrong_text   TEXT,
  correct_text TEXT,
  created_at   TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 인덱스
CREATE INDEX ON learning_sessions (user_id, created_at DESC);
CREATE INDEX ON error_logs (session_id);
CREATE INDEX ON error_logs (error_type);
```

### 5.5 에디터 입력 방식
`<Manuscript />`는 렌더링 전용이므로 입력은 별도 `<textarea>`로 처리:

```
사용자 키 입력
    ↓
hidden <textarea> (실제 DOM 포커스)
    ↓
onChange → Zustand text 상태 업데이트
    ↓
buildCursorMaps(text) → cursorCell 계산
    ↓
규칙 검사 엔진 (src/lib/rules/) → errorCells 계산
    ↓
<Manuscript text={text} cursorCell={...} errorCells={...} />
    ↓
세션 종료 시 → POST /api/sessions + POST /api/errors (배치)
```

## 6. 원고지 규칙 교육 콘텐츠 설계

### 6.1 핵심 규칙 10가지 (교육 순서)

| 순서 | 규칙 | 예시 |
|------|------|------|
| 1 | 문단 첫 칸 들여쓰기 | `□오늘은 날씨가` |
| 2 | 마침표·쉼표는 앞 글자와 같은 칸 | `다.` → 'ㅏ' 위에 '.' |
| 3 | 줄 첫 칸에 문장부호 불가 | 앞 행 마지막 칸에 같이 씀 |
| 4 | 대화 여는 따옴표 → 새 칸 시작 | `□"안녕하세요"` |
| 5 | 숫자 두 자리 → 한 칸에 병기 | `19` → 한 칸 |
| 6 | 영문 소문자 두 자 → 한 칸 | `ai` → 한 칸 |
| 7 | 제목은 중앙 배치 (상단 2칸 여백) | 1~2행 비움 |
| 8 | 의존명사 띄어쓰기 | `것, 수, 바, 만큼` |
| 9 | 보조용언 띄어쓰기 | `먹어 보다, 해 주다` |
| 10 | 자주 틀리는 맞춤법 | `됐다/됬다`, `안/않` |

### 6.2 레벨별 문제 예시

**Lv.1 — 받아쓰기 예시**
> "봄이 왔습니다. 꽃이 피었습니다."
> → 문단 들여쓰기, 마침표 위치 집중 훈련

**Lv.2 — 교정 퀴즈 예시**
> "나는 밥을먹었다. 그것은 맞는것 같다."
> → '밥을 먹었다', '맞는 것' 띄어쓰기 오류 찾기

**Lv.3 — 논술 원고지 작성**
> "환경 오염의 심각성과 해결 방안에 대해 논하시오." (400자)
> → 제목 형식, 단락 구분, 인용 따옴표

**Lv.4 — 실무 문서**
> 결재 문서, 기획안, 이메일 본문 맞춤법 교정

## 9. 리스크 및 대응

| 리스크 | 가능성 | 대응 |
|--------|--------|------|
| 한국어 맞춤법 검사 정확도 한계 | 중 | 자체 오류 사전 구축 병행, 오류 신고 기능 |
| 모바일에서 원고지 입력 UX 불편 | 높 | Phase 1은 데스크톱 전용, 이후 전용 모바일 키보드 구현 |
| `react-korean-manuscript` 패키지 미성숙 | 중 | 직접 기여 또는 포크로 필요 기능 추가 |
| Railway 무료 플랜 슬립(sleep) 이슈 | 중 | Hobby 플랜($5/월)으로 시작, 항시 가동 유지 |
| PostgreSQL 연결 수 초과 (서버리스 환경) | 중 | PgBouncer 또는 postgres.js 연결 풀 설정 (`max: 5`) |
| 교육 시장 진입 장벽 | 중 | 교사 대상 무료 파일럿 → 기관 단위 B2B 전환 |

## 10. 부록 — react-korean-manuscript 컴포넌트 활용 매핑

| 서비스 기능 | 사용 prop/유틸 |
|------------|---------------|
| 원고지 렌더링 | `<Manuscript text={text} />` |
| 커서 위치 시각화 | `cursorCell`, `cursorSide` |
| 띄어쓰기 점 표시 | `showSpaceMarks={true}` |
| 행 번호 (학습 보조) | `showRowNums={true}` |
| 오류 칸 빨간 표시 | `errorCells={new Set([3, 7, 12])}` |
| 셀 클릭 (교정 퀴즈) | `onCellClick={(cellIdx) => handleClick(cellIdx)}` |
| 페이지 분리 렌더링 | `pageIndex={currentPage}` |
| 따옴표 쌍 추적 | `dblQuoteInit`, `sglQuoteInit` |
| 커서 ↔ 문자 변환 | `buildCursorMaps(text)` |
| 셀 배열 직접 접근 | `buildFormattedCells(text)` |
